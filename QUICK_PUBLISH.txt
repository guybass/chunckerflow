═══════════════════════════════════════════════
  OPTION 1: AUTOMATED (Easiest)
═══════════════════════════════════════════════

Run the publish script:

    publish.bat

Follow the prompts. Done!

═══════════════════════════════════════════════
  OPTION 2: MANUAL (Step-by-Step)
═══════════════════════════════════════════════

1. Clean previous builds:
   ───────────────────────────────────────────
   if exist build rmdir /s /q build
   if exist dist rmdir /s /q dist
   if exist chunk_flow.egg-info rmdir /s /q chunk_flow.egg-info

2. Clean notebooks:
   ───────────────────────────────────────────
   python clean_notebooks.py

3. Run tests:
   ───────────────────────────────────────────
   pytest

4. Build packages:
   ───────────────────────────────────────────
   python -m build

5. Check distribution:
   ───────────────────────────────────────────
   twine check dist/*

6. Upload to PyPI:
   ───────────────────────────────────────────
   twine upload dist/*

   When prompted:
     Username: __token__
     Password: pypi-YOUR-TOKEN-HERE

═══════════════════════════════════════════════
  FIRST TIME SETUP
═══════════════════════════════════════════════

Install build tools:

    python -m pip install --upgrade build twine

Create PyPI account:
    1. Sign up: https://pypi.org/account/register/
    2. Verify email
    3. Create API token: https://pypi.org/manage/account/token/
    4. Save token securely

═══════════════════════════════════════════════
  TESTING ON TESTPYPI (Recommended First Time)
═══════════════════════════════════════════════

1. Upload to TestPyPI:
   ───────────────────────────────────────────
   twine upload --repository testpypi dist/*

2. Test install:
   ───────────────────────────────────────────
   pip install --index-url https://test.pypi.org/simple/ --extra-index-url https://pypi.org/simple/ chunk-flow

═══════════════════════════════════════════════
  POST-PUBLISH
═══════════════════════════════════════════════

1. Create git tag:
   ───────────────────────────────────────────
   git tag -a v0.1.0 -m "Release v0.1.0"
   git push origin v0.1.0

2. Create GitHub release:
   ───────────────────────────────────────────
   Go to: https://github.com/YOUR-USERNAME/chunk-flow/releases/new
   - Select tag v0.1.0
   - Upload dist/*.tar.gz and dist/*.whl
   - Publish

3. Bump version for next dev cycle:
   ───────────────────────────────────────────
   Edit chunk_flow/__init__.py: __version__ = "0.2.0.dev0"
   Edit pyproject.toml: version = "0.2.0.dev0"
   git commit -am "chore: bump version to 0.2.0.dev0"
   git push

═══════════════════════════════════════════════
  TROUBLESHOOTING
═══════════════════════════════════════════════

❌ "Version already exists"
   → Increment version number, rebuild

❌ "Invalid credentials"
   → Use __token__ as username
   → Paste full token as password (pypi-...)

❌ "Tests failed"
   → Fix failing tests first
   → Never publish with failing tests

❌ "Distribution check failed"
   → Ensure README.md is valid markdown
   → Check pyproject.toml syntax
